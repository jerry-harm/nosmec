# Implement mock test infrastructure for TDD tests

## Goal

Build mock infrastructure to enable proper TDD testing for nosmec's utility functions without requiring real network/relay connections. This enables isolated unit testing for functions like `GetParentPostInfo`, `ReplyToNote`, `SearchEvents`, etc.

## What I already know

- Current tests are mostly table-driven without mocks
- Functions like `GetPost`, `GetParentPostInfo`, `ReplyToNote` require `AppContext` and network access
- `testify` is NOT in the project — needs to be added
- `nostr.Pool` is a concrete struct, not an interface
- Use `uber-go/mock` (gomock) to directly mock concrete types

## Open Questions

*None remaining — using gomock with concrete types, no interface changes needed.*

## Requirements

### Phase 1: Add testify dependency
```bash
go get go.uber.org/mock/gomock
go get github.com/stretchr/testify
```

Structure:
```
testutil/

```

### Phase 3: Generate mocks with gomock
```bash
mockgen -source=../config/context.go -destination=mocks/app_mock.go
```

## Requirements

### Phase 1: Identify Mock Targets
Functions that need mocking to be testable in isolation:

| Function | Dependency | Mock Needed |
|----------|-----------|-------------|
| `GetPost` | `AppContext.Pool()` | RelayPool mock |
| `GetParentPostInfo` | `GetPost` → Pool query | Event store mock |
| `ReplyToNote` | `GetNote`, `AllWritableRelays` | Pool mock |
| `SearchEvents` | `Pool.FetchMany` | Pool mock |
| `GetFullProfile` | `Pool.QuerySingle` | Pool mock |
| `ListDMConversations` | `Pool.SubscribeMany` | Pool mock |

### Phase 2: Define Mock Interfaces
Create interfaces for consuming structs:

```go
// AppContext interface for testing
type AppContextInterface interface {
    GetMySecretKey() (nostr.SecretKey, error)
    ListRelays() []config.Relay
    AllReadableRelays() []string
    AllWritableRelays() []string
    Pool() *nostr.RelayPool
    // ... other methods used by utils
}
```

### Phase 3: Implement Test Helpers
Create test utilities:

```go
// testutil/mock_app.go
type MockAppContext struct {
    mock.Mock
}

// testutil/mock_pool.go
type MockPool struct {
    mock.Mock
}
```

### Phase 4: Write TDD Tests with Mocks
For each function, follow TDD cycle:
1. RED: Write failing test with mocks
2. GREEN: Write minimal implementation (or add nil checks as needed)
3. REFACTOR: Clean up
